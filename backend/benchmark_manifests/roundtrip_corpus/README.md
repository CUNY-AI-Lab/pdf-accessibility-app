# Round-Trip Corpus

Each `*.roundtrip.json` file defines one gold-to-stripped verification case.

Each manifest should also declare:

- `document_family`
  - examples: `manual`, `technical_report`, `conference_paper`, `form`
- `product_priority`
  - `primary` for document families aligned with the main product target
  - `secondary` for stress suites that matter but are not the core baseline
- `workflow_tags`
  - short capability tags such as `toc`, `bookmarks`, `front_matter`, `forms`, `tables`

Those fields feed the corpus-level non-regression matrix so the workflow is judged across document families, not just case-by-case.

Golds in this corpus must be:

- independently produced, not generated by this app
- validated explicitly against `ua1`
- manually spot-checked for practical accessibility

Do not use local workflow outputs or "looks tagged" PDFs as gold sources.

You can run the corpus in parallel at the case level:

```bash
cd backend
PYTHONPATH=. uv run python scripts/roundtrip_corpus_benchmark.py --workflow-profile assistive-core --jobs 4
```

The runner now defaults to `4` jobs. Reduce it if shared services such as `docling-serve` or remote LLM endpoints become the bottleneck.
